## План наполнения протокола

По мере уточнения требований сюда переносятся (без дублирования в письме заказчику):

1. **Модель данных** — VisitSession, Chat, Message, Assignment, Department, Placement
2. **RealTime API** — события WebSocket/SSE для АРМ и виджета
3. **REST API** — администрирование, история, отчёты
4. **Контракт суфлёр ↔ чат** — DialogEvent, SuflerHint, OperatorFeedback
5. **Адаптеры мессенджеров** — Telegram, Viber, VK, OK (Фаза 3)
6. **Нефункциональные требования** — нагрузка, отказоустойчивость, ИБ, аудит
7. **Traceability** — пункты Приложения 1 п. 4.4 → UC → тест-кейсы
