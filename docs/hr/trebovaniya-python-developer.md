## О проекте

On-prem AI-платформа в **закрытом контуре банка** (без выхода inference в интернет):

| Модуль | Назначение |
|--------|------------|
| **AI Hub** | Оболочка, RBAC, админ-центр |
| **LLM + RAG** | Индексация базы знаний СУЗ (~1700 статей), подсказки операторам |
| **Контакт-центр** | Суфлёр (телефония + онлайн-чат), отчётность |
| **ИИ-ассистент** | Внутренний чат для сотрудников банка |
| **Интеграции** | 1С-Битрикс (СУЗ), Oktell (телефония), Active Directory (LDAPS), SIEM (Kaspersky KUMA) |

**Текущее состояние кода:** прототип Django (`dashboard/app/`) + ASR-сервис (`recognizer/main.py`); основная разработка — greenfield по [единому ТЗ](../modules/ai-hub/tz-unified-v1.2.md).

**Нагрузочные ориентиры:** ~6000–8000 звонков/сутки, p95 подсказки ≤2 с, ~70 параллельных звонков, синхронизация RAG с СУЗ — 1–2 мин после публикации статьи.

## Обязательные требования

### Python и backend

- Коммерческий опыт **Python 3.10+** от **2 лет**
- **Django** (REST API, ORM, миграции, admin) — уверенный уровень
- **Async Python:** asyncio, WebSocket-сервисы (Channels / websockets / Starlette)
- **PostgreSQL:** проектирование схем, индексы, транзакции, миграции
- **REST API:** проектирование контрактов, валидация, пагинация, версионирование
- **Streaming:** SSE или WebSocket для потоковой выдачи LLM-ответов
- **Очереди и фоновые задачи:** Celery / RQ / аналоги; паттерны retry, idempotency, outbox
- **Тестирование:** pytest, unit/integration-тесты для API и бизнес-логики
- **Git:** feature-branch workflow, code review

### RAG / LLM

- Понимание архитектуры **RAG**: chunking, embeddings, vector search, reranking
- Опыт или готовность быстро освоить **LangChain / LlamaIndex / собственный pipeline**
- Работа с **векторными БД** (pgvector, Qdrant, Milvus, Chroma — любой из стека)
- Инкрементальная индексация: webhook-приём, дедупликация, reconciliation
- Метрики качества: latency, hallucination rate, citation/source tracking

### Интеграции и enterprise

- **Webhook consumers:** HMAC-подписи, mTLS, allowlist, обработка дубликатов
- **LDAP/LDAPS:** аутентификация через AD, маппинг групп → роли (RBAC)
- **Audit logging:** структурированные события, экспорт в SIEM
- Понимание **OWASP Top 10 for LLM** (prompt injection, data leakage между индексами)
- Опыт **on-prem / closed contour** деплоя (без внешнего интернета для inference)

### DevOps (базовый уровень)

- Docker, docker-compose; понимание CI/CD
- Linux-серверы, nginx/reverse proxy, TLS
- Мониторинг: логи, метрики, health checks

## Зона ответственности

**Python (~80%):**

1. RAG-пайплайн: ingestion СУЗ → chunking → embeddings → vector index `cc_production`
2. LLM-оркестратор: промпты (`sufler_cc`, `assistant_bank`), streaming, citations
3. Webhook-сервис СУЗ (модель B): INT-01…10, outbox, reconciliation
4. Интеграция Oktell: WebSocket `phoneevent_*`, dual-leg ASR → подсказки
5. Backend онлайн-чата: сессии, очереди, операторский ARM API
6. ИИ-ассистент API: `/api/v1/assistant`, SSE, KB management
7. LDAPS + RBAC (13 ролей), audit → KUMA
8. PostgreSQL, нагрузочное тестирование, formal acceptance (SUF-T, CHAT-T, ASS-T, INT-T)

**.NET (~10–20%):**

- Исправление дефектов и мелкие доработки в смежном проекте
- Сопровождение API-контрактов между .NET-сервисами и Python-контуром
- Code review и консультации по integration points
