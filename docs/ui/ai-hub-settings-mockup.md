## Принципы компоновки

| Параметр | Значение |
| -------- | -------- |
| Оболочка | Полноэкран; вход из меню **≡** панели AI Hub |
| Sidebar | ~240 px, группы с заголовками; видимость пунктов по RBAC |
| Рабочая область | Breadcrumbs · заголовок H1 · контент · sticky footer с «Сохранить» |
| Embed Studio | Промпты и редактор сценариев — **внутри** Hub, не отдельная карточка «Studio» во «Внешних системах» |
| Профили LLM | `assistant_bank` (ассистент) и `sufler_cc` (КЦ) — **раздельные** preset параметров §3.3 |

**Токены:** как в [ai-hub-panel-mockup.md](ai-hub-panel-mockup.md) — `#0D3880`, `#E31E24`, radius 8 px.

## Термины (не смешивать в UI)

| Термин в интерфейсе | Что это | Где настраивается |
| ------------------- | ------- | ----------------- |
| **Конфигурация LLM** | Dashboard 7 слоёв: параметры, промпты, навыки, KB, QU, политики, тест | Hub → landing + deep links |
| **Промпт** | System / task instruction для LLM | «Промпты ассистента» или вкладка «Промпты» в редакторе сценария |
| **Параметры модели (§3.3)** | Temperature, chunk, семантика RAG — **не** промпт | «Параметры модели LLM» |
| **Навык / capability** | Что LLM **может** делать (RAG, RPA, doc…) | «Навыки и инструменты» + детальные экраны |
| **Skill-группа чата** | Маршрутизация диалогов операторов | **Online Chat admin** — не Hub |
| **Шаблон ответа оператора** | Готовый текст для вставки в чат | **Online Chat admin** — не Hub |
| **Диалоговый сценарий КЦ** | Flow «Услуга» + промпты `sufler_cc` | «Редактор сценариев» |

## Навигация (полный sidebar)

Группы и порядок — для слайда **I-2** и Figma.

```
≡ Центр настроек AI Hub
├── ОБЩЕЕ
│   ├── Подразделения и scope
│   └── Журнал изменений
├── АССИСТЕНТ
│   ├── Конфигурация LLM          ← landing (dashboard)
│   ├── Параметры модели LLM
│   ├── Промпты ассистента
│   ├── Навыки и инструменты
│   ├── Базы знаний
│   ├── Понимание запросов (QU)
│   ├── Источники данных
│   ├── Инструменты ассистента    ← RPA, шаблоны, SQL
│   └── Мониторинг ассистента
├── СУФЛЁР / КЦ
│   ├── Конфигурация LLM КЦ
│   ├── Параметры модели (КЦ)
│   ├── Редактор сценариев
│   ├── Тест сценария
│   ├── Сценарии суфлёра          ← привязки отделов
│   └── Политики суфлёра
├── ДОКУМЕНТЫ (OCR)
│   ├── Типы документов
│   └── Экспорт документов
└── ССЫЛКИ
    └── Внешние системы
```

**RBAC (упрощённо):** Админ KB видит блок «АССИСТЕНТ»; Админ сценариев / КЦ — «СУФЛЁР / КЦ»; Админ OCR — «ДОКУМЕНТЫ»; Аудитор — read-only везде, где разрешено.

В Canvas — **переключатель «Демо роль»** в шапке: влияет на права редактирования, **не скрывает** пункты sidebar.

## Где сценарии КЦ в Центре настроек

**Не** во «Внешних системах» и **не** в блоке «Ассистент». Диалоговые сценарии (`sufler_cc`) — группа sidebar **«СУФЛЁР / КЦ»**:

| Пункт sidebar | Screen id | Что делает |
| ------------- | --------- | ---------- |
| **Редактор сценариев** | `scenario_editor` | Реестр ≥50 · карта · промпты · публикация (§II.6.1) |
| **Тест сценария** | `scenario_test` | Sandbox + отчёт 4.5.2.8 (слайд II-4) |
| **Сценарии суфлёра** | `scenario_bindings` | Привязка отделов → `scenario_id` |
| **Конфигурация LLM КЦ** | `llm_config_cc` | Dashboard; слои 1–2 ведут в редактор |

**Не путать:**

| Объект | Где |
| ------ | --- |
| Диалоговый сценарий КЦ | **Редактор сценариев** |
| RPA для ассистента | **Инструменты ассистента** |
| Skill-группа оператора чата | **Online Chat admin** (внешняя ссылка) |
| Шаблон ответа оператора | **Online Chat admin** |

Открой Canvas → прокрути sidebar до **«СУФЛЁР / КЦ»** → **«Редактор сценариев»**.

## Конфигурация LLM — 7 слоёв

Единая концепция для заказчика; два dashboard — по профилю.

| Слой | Название | Ассистент → раздел Hub | КЦ → раздел Hub |
| ---- | -------- | ---------------------- | --------------- |
| **0** | Параметры модели | Параметры модели LLM | Параметры модели (КЦ) |
| **1** | Instructions | Промпты ассистента | Редактор сценариев → Промпты |
| **2** | Capabilities / Flow | Навыки и инструменты | Редактор сценариев → Карта |
| **3** | Knowledge | Базы знаний | Базы знаний / webhook СУЗ (read-only) |
| **4** | Understanding | Понимание запросов | QU + Политики суфлёра (пороги) |
| **5** | Guardrails | Фильтры + контекст истории | Политики суфлёра |
| **6** | Test & Publish | «Проверить конфигурацию» | Тест сценария + отчёт |

### `model_params` — Параметры модели LLM (§3.3)

**§III.11.1** · **§II.6.2.1** (профиль КЦ) · переключатель **Ассистент | КЦ (sufler_cc)**

```
Параметры модели LLM · профиль: Ассистент (assistant_bank)

Генерация                    RAG / индексация
─────────────────            ─────────────────────────
Температура        [====●--] 0.35   Chunk size      [512  ] симв.
Max ответа         [1200  ] симв.   Overlap         [64   ] симв.
                             Семант. порог контекста  [72%] ───
                             Детермин. ответ из KB    [95%] ───

ℹ Порог QU 5.2.12 → «Понимание запросов» (не дублировать с 3.3.5)

[Сброс к дефолту]  [Сохранить]  [Тест с параметрами]
```

| Блок | Поля | Прил.1 |
| ---- | ---- | ------ |
| **Генерация** | Temperature (slider 0–1); Max длина ответа; presets краткий/стандарт/развёрнутый | 3.3.1–3.3.2, 3.4 |
| **RAG / индексация** | Chunk size; Overlap; Семант. порог в контекст (%); Порог детерминированного ответа из KB (%) | 3.3.3–3.3.6 |
| **Read-only** | Контекстное окно ≥8200; выбор модели LLM per модуль | 3.1, 3.2 |

**Preset КЦ (`sufler_cc`):** temperature 0.1–0.25; max ответ ≤500 симв.; семантика строже.

**Footer:** [Сброс к дефолту платформы] [Сохранить] [Тест с параметрами]

**Cross-links:** QU 5.2.12 → `qu_admin`; политики релевантности КЦ → `sufler_policies`.

## Сводка: промпты vs параметры vs навыки

```
Пользователь задаёт вопрос
        │
        ▼
┌─ §3.3 Параметры модели ─── temperature, chunk, семантика RAG
├─ Промпты ───────────────── system/task instructions
├─ Capabilities ──────────── какие инструменты доступны
├─ KB + QU ───────────────── что найти и когда отвечать
└─ Guardrails ────────────── фильтры, история, пороги суфлёра
```
