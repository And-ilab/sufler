# Политика контроля конфиденциальной информации

**Версия:** 1.0 · **Дата:** 2026-06-22 · **Проект:** ПО на базе искусственного интеллекта для банковских процессов · **Договор:** № 14-03/2026

---


## 1. Контекст и назначение

### 1.1. Контекст

Конфиденциальная информация заказчика и ООО «ГС Ритейл» при выполнении договора № 14-03/2026 требует особого режима обращения. Заказчик при оценке киберустойчивости указал на отсутствие процесса контроля утечки конфиденциальной информации.

Настоящая политика дополняет раздел о классификации данных в politika-informacionnoj-bezopasnosti.md и устанавливает **конкретные меры** по предотвращению несанкционированного раскрытия.

### 1.2. Назначение

Определить правила обращения с конфиденциальной информацией, допустимые каналы передачи и меры контроля утечек.

### 1.3. Область действия

Вся конфиденциальная и ограниченного распространения информация, обрабатываемая при проекте «{PROJECT}».

Политика преследует цель **исключить** передачу конфиденциальных данных через не контролируемые каналы и **обеспечить прослеживаемость** решений о допустимых способах обмена информацией с {CUSTOMER}.

## 7. Меры контроля утечек

### 7.1. Организационные меры (действуют)

1. **Классификация и маркировка** — руководитель проекта обеспечивает маркировку документов и каталогов (префикс «Конфиденциально» / «Для ОАО «АСБ Беларусбанк»»).
2. **Запрет мессенджеров** — передача конфиденциальной информации через Telegram, Viber и аналоги **запрещена**.
3. **Контроль носителей** — копирование на личные USB и внешние диски только с письменного разрешения координатора ИБ.
4. **Доступ по ролям** — принципы need-to-know и least privilege (см. политику ИБ, раздел 7).
5. **Аудит доступов** — администратор ИТ ежеквартально сверяет список лиц с доступом к данным проекта с актуальным составом команды (совместно с HR).
6. **Удаление по завершении** — по окончании договора данные банка удаляются или возвращаются по акту; координатор ИБ контролирует исполнение.

### 7.2. Технические меры (перспектива)

Внедрение DLP-средств (мониторинг каналов, блокировка передачи) запланировано в дорожной карте 2026–2027. До внедрения DLP действуют организационные меры и выборочный аудит.

## 8. Обмен информацией с банком (вопрос 31 анкеты)

Обмен информацией ограниченного распространения с ОАО «АСБ Беларусбанк» осуществляется:

- через **корпоративную электронную почту** с домена ООО «ГС Ритейл»;
- через **BelVPN** и согласованные ресурсы банка;
- при требовании банка — через **СКЗИ** по отдельному согласованию.

Инициатор нового канала — руководитель проекта; согласование — координатор ИБ и представитель банка.
